# Platform Setup Public Notes

This page tracks the public-safe automation plan for SIGAudit Global.

The automatic footprint workflow has two layers:

1. Publish public-safe bilingual content to this repository.
2. Reuse the same approved public content in professional platforms through official platform integrations only.

No private information belongs in this repository. Do not add employer names, client names, internal systems, private links, real datasets, unpublished biomedical details, or confidential code.

## Platform plan

- GitHub: primary automatic channel.
- LinkedIn: use only official developer integration when available.
- YouTube: use only official upload integration, starting with private visibility.
- Medium: optional because API support is limited.
- Other platforms: keep draft-only until an official permitted integration exists.

## Safe content scope

Priority topics:

- GIS and ArcGIS data quality.
- Migration QA/QC.
- Oracle, Python, C# and Excel reporting automation.
- Technical documentation.
- Utility GIS workflows.
- SIGAudit Global digital products and services.

Biomedical/Conn content must remain public, educational, preliminary, non-clinical and free of treatment claims.

## Operational note

Sensitive platform access values must be stored outside the repository, in the platform-approved protected settings for automation. They must never appear in commits, issues, README files, screenshots or chat transcripts.
